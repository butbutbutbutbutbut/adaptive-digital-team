# 中文内容

本目录是 Adaptive Digital Team（ADT）的中文阅读入口。

ADT 是一个帮助 AI 入门者、独立开发者和个人创作者，与 AI Agent 建立可靠协作工作流的公开提示词与治理仓库。它通过任务授权、状态记录、写入范围、候选检查和人工最终确认，降低跨窗口、跨设备和长期项目协作中的上下文丢失与权限失控风险。

> 仓库的正式运行状态以根目录 `PROJECT_STATE.md` 和当前 `main` 分支为准。

## 项目结构说明

| 路径 | 主要内容 |
|---|---|
| `.github/workflows/` | GitHub Actions 自动化流程，包括验证、PR 资格检查和候选历史检查。 |
| `docs/` | 项目架构、公开边界、治理设计和其他说明文档。 |
| `protocols/` | ADT 的治理协议、任务流程、状态转换和权限门禁规则。 |
| `schemas/` | 项目状态、任务记录和验证数据使用的结构定义。 |
| `scripts/` | 静态验证、状态检查和其他自动化脚本。 |
| `tests/` | 自动化测试，包括治理约束、候选历史和初学者启动器测试。 |
| `中文内容/` | 面向中文用户的简要说明和后续中文文档入口。 |
| `AGENTS.md` | AI Agent 必须遵守的角色、权限、执行顺序和治理规则。 |
| `BOOTSTRAP.md` | AI 首次读取仓库时使用的启动顺序与上下文加载规则。 |
| `PROJECT_STATE.md` | 当前项目状态、活动任务、授权范围和运行基线。 |
| `README.md` | 项目公开介绍、适用场景、使用入口和当前开发状态。 |

## 当前重要文件

### `README.md`

适合第一次接触 ADT 的用户阅读，包含项目用途、适用人群、A/B/C 三种启动方式、推荐运行方式和当前开发进度。

### `AGENTS.md`

规定 AI Agent 的行为边界，包括谁可以授权任务、哪些文件允许修改、什么情况下必须停止、候选/审计/Ready/Merge 的区别，以及初学者入口与高级控制包的分流方式。

### `PROJECT_STATE.md`

记录当前仓库的正式状态。AI 不应仅依赖聊天记录判断项目进度，应同时核验该文件和仓库事实。

### `protocols/`

保存具体治理协议。当前包括候选执行、权限门禁、历史保护和初学者启动路由等规则。

其中 `BEGINNER_BOOTSTRAP_ROUTER.md` 是初学者首次接触 ADT 时的交互协议。

### `tests/`

用于验证协议是否真正落实，不仅停留在文档描述中。

## 建议阅读顺序

1. 根目录 `README.md`
2. 本文件
3. 根目录 `AGENTS.md`
4. 根目录 `PROJECT_STATE.md`

普通使用者通常不需要完整阅读所有协议和测试文件。

## 使用提醒

- 只使用 A 或 B 时，不需要 GitHub 账号，也不需要创建分支或 PR。
- 选择 C 时，应提供自己拥有或管理的项目仓库。
- ADT 公共仓库默认只作为规则和上下文来源，不是外部用户的项目工作区。
- 跳过初学者引导不等于获得仓库写入、Ready 或 Merge 权限。
- 真实密码、令牌、私钥和内部配置不得写入公开仓库。
