# 中文内容

本目录是 Adaptive Digital Team（ADT）的中文阅读入口。

ADT 是一个**治理工具系统和控制平面**——一套协议、验证器和可复用的检查机制，帮助人类和 AI Agent 在多智能体项目中安全协作。它不是运行时、不是调度器、也不是单一 AI 人格。

> 仓库的正式运行状态以根目录 `PROJECT_STATE.md` 和当前 `main` 分支为准。

## 你是人类还是 AI？

- **人类 / 普通访问者** → 继续往下读，从 A/B/C 选择使用场景
- **交互式 AI / Agent** → 立即跳转到 [BOOTSTRAP.md](../BOOTSTRAP.md) 激活协议

---

## A / B / C 快速开始

完整路由定义见 [protocols/BEGINNER_BOOTSTRAP_ROUTER.md](../protocols/BEGINNER_BOOTSTRAP_ROUTER.md)。

**菜单自动跳过**：第一条消息已包含明确任务、附件、仓库链接或控制包时，自动跳过菜单。

### A｜直接开始

直接告诉安鼎想完成什么。建议包含：想得到什么结果、已经有哪些信息、希望以什么形式交付。此模式不涉及仓库操作。

### B｜我会上传文件

把文件上传到当前对话，然后告诉安鼎需要修改、分析还是整理。可以一次上传多个文件，不需要上传到 GitHub。

### C｜连接我自己拥有或管理的项目仓库

提供自己拥有或管理的项目仓库链接，然后选择只读分析或允许创建候选变更。最终提交和合并仍由你确认。

输入 **返回模式选择** 可随时重新显示菜单。

---

## ⚠️ AI 激活说明

交互式 AI 读取本仓库后自动进入 ADT 协议，获得「安鼎」界面身份（ANDING_INTERFACE）。这是设计目标，不是漏洞。

但请注意：**进入协议 ≠ 获得写入 / Ready / Merge / Control 权限**。完整激活条件见 [BOOTSTRAP.md](../BOOTSTRAP.md)。

## 项目结构

| 路径 | 主要内容 |
|---|---|
| `.github/workflows/` | GitHub Actions 自动化流程，包括验证、PR 资格检查和候选历史检查。 |
| `docs/` | 项目架构、公开边界、治理设计和其他说明文档。 |
| `protocols/` | ADT 的治理协议、任务流程、状态转换和权限门禁规则。 |
| `schemas/` | 项目状态、任务记录和验证数据使用的结构定义。 |
| `scripts/` | 静态验证、状态检查和其他自动化脚本。 |
| `tests/` | 自动化测试，包括治理约束、候选历史和初学者启动器测试。 |
| `中文内容/` | 面向中文用户的简要说明和中文文档入口。 |
| `AGENTS.md` | AI Agent 的核心行为边界与规范索引。 |
| `BOOTSTRAP.md` | 协议激活入口——读取即激活 ADT，定义 Agent 读取顺序。 |
| `PROJECT_STATE.md` | 当前项目状态、活动任务和运行基线。 |
| `README.md` | 项目公开介绍、使用入口和当前开发状态。 |

## 建议阅读顺序（人类）

1. 根目录 `README.md`
2. 本文件
3. 根目录 `AGENTS.md`（如需了解 Agent 行为规则）
4. 根目录 `PROJECT_STATE.md`

普通使用者通常不需要完整阅读所有协议和测试文件。

## 使用提醒

- 只使用 A 或 B 时，不需要 GitHub 账号，也不需要创建分支或 PR。
- 选择 C 时，应提供自己拥有或管理的项目仓库。
- ADT 公共仓库默认只作为规则和上下文来源，不是外部用户的项目工作区。
- 跳过初学者引导不等于获得仓库写入、Ready 或 Merge 权限。
- 真实密码、令牌、私钥和内部配置不得写入公开仓库。
